# Decision Log

## 2026-04-27: House-view ingest stays pure; data augmentation moves to consumer-skill JIT
**Why:** End-to-end validation against real CIO docs surfaced two problems with auto-gap-fill at ingest. **(1) Functional:** the fold helpers were written against assumed response shapes; the live API nests differently and returns sector positioning as narrative prose, not structured maps. Result: zero usable suggestions on live data despite passing unit tests against synthetic fixtures. **(2) Architectural:** even with helpers fixed, silently injecting current-data values into the saved view broke the trust contract — when the saved view mixes "what the bank said" with "what current data said today," compliance can no longer answer "is this our bank's view or a derived view?" cleanly. The pivot: **saved house view stays pure (bank's view + uploader confirmation only); current-data augmentation happens at consumer-skill use time, with provenance recorded per consuming artifact, never folded back into the saved view.**
**Impact:**
- Gap-fill detection + suggestion modules ship as pure functions (`gap_detect`, `gap_suggest`) with hash-chained audit + compliance-export modules. Live-API-shape regression tests pin behavior against captured response fixtures — caught the unit-tests-passed-but-production-failed gap.
- `/parallax-load-house-view` ingest no longer carries a gap-fill review step. Save flow writes `view.yaml` + `prose.md` + `provenance.yaml` + `audit.jsonl` only; current-data overlays never appear in the saved view.
- `/parallax-portfolio-builder` is the first consumer wired for JIT augmentation: opt-in `--augment-silent` flag triggers `gap_detect` + `gap_suggest` against silent thesis-relevant dimensions; output table carries a per-row Tilt Source column tagging each multiplier as bank-view-sourced, current-data-sourced (with tool + as-of date), or neutral; audit log records the flag value + augmented dimensions + skipped dimensions.
- Five other consumer skills (rebalance, thematic-screen, morning-brief, client-review, explain-portfolio) follow the same pattern in fast-follow PRs.
- Worked-example artifact (`examples/worked-example.md` + `.py`) is reproducible and demonstrates all three source-tag types. JIT operational policies (staleness, flag persistence) are documented in `examples/jit-policies.md`. Operator verification posture is in `examples/testing-posture.md`.
**Verification status — honest read:** Module tests pass. SKILL.md changes (consumer-skill instructions executed by Claude at runtime) need a fresh-session walkthrough to runtime-verify, since the runtime caches SKILL.md content at session start. The pre-go-live protocol is two fresh-session invocations: ingest a CIO doc and confirm the saved-view artifacts land correctly; then run a portfolio-builder invocation with `--augment-silent` and confirm the per-row Tilt Source column appears, the saved view stays unchanged, and the audit chain extends with the augmentation provenance.
**Flip conditions:** (a) live-session verification surfaces a SKILL.md gap → follow-up commit before external demo; (b) regulators require the augmentation provenance to live on the saved view rather than per-artifact → revisit with a sibling overlay file design; (c) the JIT flag becomes the de-facto operator default in observed usage → consider flipping the default and adding an explicit-pure flag.
**Alternatives:** [DROP] Keep gap-fill at ingest with the fold-helper fix only — rejected; the trust-contract problem persists even with functional fixes. [DROP] Move to a post-hoc augment surface as the only path — rejected; the operator instinct is to augment at the point of need, not after-the-fact against the saved view. [DROP] A view-side consistency-warning surface — largely obsoleted by the pivot since the saved view no longer mixes sources. [DROP] Wire all six consumer skills in one milestone — rejected as scope creep; the first consumer is the highest-leverage validation case.

## 2026-04-26: House-view ingest gap-fill — suggest-with-per-field-accept
**Why:** External CIO macro reports are often partial (sectors only, silent on regions; macro regime only, silent on factor tilts). The existing `schema.yaml` "omitted tilt defaults to neutral" rule conflates "uploader didn't take a view" with "uploader was deliberately neutral." Added an optional ingest step that detects silent dimensions, queries the data-driven macro view for candidate values, and presents them to the uploader for per-field accept/reject — preserving the uploader-confirms-every-value invariant rather than auto-merging.
**Impact:** New Step 2.5 in `/parallax-load-house-view` runs between extraction and the confirmation gate when the source is a document (skipped for wizard mode). New `--fill-gaps` operational mode re-runs the same flow against an already-saved view as post-hoc enrichment. Eligible dimensions are `macro_regime`, `factors`, `regions`, `sectors`; the existing Phase 0 boundary on pillars (encoding-only) and the existing CIO-judgment dimensions (themes, styles, excludes) is preserved end-to-end via an allowlist guard tested at planner level. New provenance classification `parallax_data_fill` added to `provenance.yaml` taxonomy in schema.yaml; carries source tool + call args + verbatim snippet + `data_as_of` so `--why <tilt-path>` can surface the data origin and warn that the value reflects live data captured at ingest (may drift). Three new audit actions in loader.md §6.1: `gap_fill_suggested` / `gap_fill_accepted` / `gap_fill_rejected`, each appended through the existing hash-chained `audit_chain.append_entry()`. Detection uses word-boundary regex against prose + extraction_notes (avoids false positives like "us" matching inside "cautious") and a small alias map for common surface forms. Suggestion sourcing is a pure module; the skill runtime makes the MCP calls and feeds responses back. Per-field UX is a single multi-select prompt with a batch review table (sequential prompts would be unusable at the 27-region schema scale). Security pass surfaced two file-permission gaps (audit.jsonl + provenance.yaml at-rest perms) — both fixed with explicit 0600 enforcement in `audit_chain` and the SKILL.md write step. Code-cleaner pass surfaced two dead imports — removed. **74/76 tests pass** in the house-view subtree (43 net new across unit + smoke; 2 pre-existing failures unrelated). **Flip conditions:** (a) data-driven values age stale faster than uploaders notice → ship `--refresh-data-fills` (uses `data_as_of` recorded today); (b) stronger epistemic separation between discretionary view and derived signal becomes desirable → fall back to overlay-only by promoting accepted suggestions to a sibling `data_view.yaml` instead of folding into `view.yaml` (per-field accept UX stays, only storage location changes); (c) macro coverage expands beyond today's market list → extend the region map; (d) low-confidence override threshold (0.4) needs tuning → single constant in `gap_detect.LOW_CONFIDENCE_THRESHOLD`; (e) telemetry exposes structured per-factor regime scores → swap the regime-derived factor mapper for a direct one.
**Alternatives:** [DROP] Auto-merge / harmonize into one unified view.yaml — most ergonomic but blurs CIO opinion vs derived signal and crosses the existing "encoding-only, not auto-applied" guardrail. [DROP] Overlay-only via a separate `data_view.yaml` — preserved as flip-condition (b); rejected for v1 because pushes work to every downstream consumer and the per-field provenance class already preserves separation. [DROP] Coverage report only (emit "silent on X, Y" and defer fill to a separate skill invocation + manual re-ingest) — defers the hard problem and generates no audit-trail provenance for the data-driven layer. [DROP] Sequential per-gap prompting — unusable at 27-region scale. [DROP] Treat all 0/null as gaps without prose-mention check — would flood the uploader with prompts on intentionally-neutral fields; absence ≠ gap is the load-bearing distinction. [DROP] Auto-apply when extractor confidence is very low without uploader confirmation — would break the uploader-confirms-every-value invariant. Full reasoning + per-finding security audit detail kept locally.
