## OWASP Findings Detail (supporting SEC-12) — PR#35 surface (skill renames + frontmatter migration + spec-validate.py + build/lint scripts)

**A01 Broken Access Control:** No new access control surface. spec-validate.py reads operator-supplied directories as argv — no access control layer needed. build-skills.sh/check-registration.sh/coverage-lint.sh are operator-run local tools with no network exposure. PASS.

**A02 Cryptographic Failures:** No new sensitive data stored or transmitted. spec-validate.py processes YAML frontmatter in memory and prints results; no persistence. Build scripts write .skill zip packages to ~/Downloads/ — operator-controlled local filesystem. No sensitive data involved in any new surface. PASS.

**A03 Injection:**

- spec-validate.py (new): Uses `yaml.safe_load(parts[1])` on SKILL.md frontmatter (confirmed line 41). No `yaml.load()`, no `eval()`, no `exec()`, no `subprocess`, no file writes. Processes operator-supplied skill directories as Path objects — no shell expansion. PASS.

- build-skills.sh line 107: Unquoted command substitution `python3 ./_parallax/scripts/spec-validate.py $(ls -d */ | while read -r d; do [[ -f "$d/SKILL.md" ]] && echo "$d"; done)` performs word-split on directory names. All current skill directory names are `parallax-[a-z0-9-]+` or `translate-[a-z0-9-]+` — no spaces, no special shell characters. Not a real injection risk in current state. INFORMATIONAL — if a skill directory with spaces or a name containing shell metacharacters were ever introduced, this substitution would split incorrectly. Recommend quoting via `mapfile`/xargs or passing via null-delimited read. No remediation required now.

- coverage-lint.sh: `sed -n "${start},${line_num}p"` uses integer variables derived from `grep -n` output — $start and $line_num are always integers from arithmetic. No command injection. PASS.

- check-registration.sh: `s=${d%/}` and `$s` are basename-level variables from `for d in */` — all operator-repo directory names, no user or network input. PASS.

- install.sh: All variable expansions properly double-quoted (`"$skill_dir"`, `"$target"`, `"$SKILLS_DIR"`). `cp -r "$skill_dir"*` uses quoted path. No injection surface. PASS.

- white-label validator.py subprocess (pre-existing, unchanged): `subprocess.run(["npx", "-y", "@google/design.md", "lint", temp_path, "--format", "json"], ...)` uses list form (no `shell=True`). temp_path is `tempfile.NamedTemporaryFile().name` — system-controlled, not user-provided. INFORMATIONAL (unchanged from prior audits).
