## OWASP Findings Detail (supporting SEC-12)

**A01 Broken Access Control:** No access control surface. No endpoints, no routes, no user roles. PASS.

**A02 Cryptographic Failures:** No sensitive data stored. Ed25519 public key only in trusted_keys.json (test-only, labeled). No deprecated hash algorithms. PASS.

**A03 Injection:**
- SQL injection: no SQL queries in the codebase. PASS.
- Command injection: subprocess calls in validator.py (lines 472, 497) use list-form arguments (["npx", "--version"] and equivalent) with no user-supplied data interpolated; shell=True absent throughout. PASS.
- New surface — render_gate.py: pure re + sys.stdin.read() + sys.stdout.write(). No subprocess, no eval, no exec, no file writes. stdin data flows through regex match only; regex patterns tested against adversarial inputs (5000-char hash string, deeply nested bold markdown) — all under 0.5 ms, no catastrophic backtracking. PASS.
- YAML injection: yaml.safe_load used at loader.py. No yaml.load() found. PASS.

**A04 Insecure Design:** No threat model present. Recommend /threat-model for the white-label onboarding flow (external URL fetching, partner config parsing) before expanding that surface. LOW priority — no deployed attack surface today.

**A05 Security Misconfiguration:** No server to misconfigure. No debug mode, no default credentials surface. PASS.
- New surface — SKILL.md render gate step: mktemp with XXXXXX suffix (6 random chars from /dev/urandom entropy pool on macOS/Linux) produces a non-predictable filename. The SKILL.md explicitly prohibits a fixed path (/tmp/pcheckup_draft.md) and documents the world-writable-/tmp symlink hazard as the reason. Holdings data flows as stdin after the temp file is written; it is never interpolated into the shell command string. No TOCTOU window: the file is written, piped into render_gate.py as stdin by shell redirection (< "$DRAFT"), and then removed — a read-after-close race on the temp file cannot affect the gate's output because the gate reads from its own stdin file descriptor, not from the path. PASS.

**A06 Vulnerable Components:** No package.json or requirements.txt at repo root. Python helpers import stdlib only (render_gate.py: re, sys). Third-party deps (yaml, jsonschema, PIL, matplotlib, pdfplumber, scrapling) are operator-installed, not bundled. No lock files to audit. INFORMATIONAL — dependency audit remains the operator's responsibility at install time.

**A07 Authentication Failures:** No auth surface. PASS.

**A08 Software and Data Integrity:** No CI/CD pipeline. No webhook handlers. PASS.

**A09 Security Logging:** Python helpers use standard logging module. render_gate.py has no logging calls — it is a pure stdin→stdout transformer. No secrets logged anywhere. PASS.

**A10 SSRF:** web_pdf.py fetches operator-supplied URLs with a 15s timeout and 5 MB read cap. No server process means no server-side SSRF risk. render_gate.py makes no network calls. INFORMATIONAL — acceptable risk for a local CLI skill.

**Attacker Perspective:**
1. Most sensitive data: partner branding configs in ~/.parallax/client-branding/config.yaml. Protected by filesystem permissions on the operator's machine — outside this repo's control.
2. render_gate.py attack surface: an adversary who can supply malicious stdin content (i.e., the orchestrating model's output) can at most cause the gate to return that content unchanged (fail-safe path) or anchor at an unexpected line. No code execution path exists. The ReDoS surface is minimal — anchors are header-form only (^#{1,4}), not open-ended repetition on variable-length content.
3. Temp file hardening: mktemp prevents the symlink-squatting attack that a fixed /tmp path would allow. The fix is sound.
4. No replay, no webhooks, no payment flows in scope.
