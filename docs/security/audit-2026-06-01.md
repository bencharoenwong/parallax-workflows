## OWASP Findings Detail (supporting SEC-12) — merged surface (eval substrate + portfolio-checkup render gate + white-label-onboard refactor)

The white-label loader reads from a fixed operator-local path (~/.parallax/client-branding/config.yaml); no user-role or endpoint involved. The eval harness is operator-only CLI. PASS.

**A02 Cryptographic Failures:** No new sensitive data stored or transmitted by any merged surface. The white-label branding config written by the onboard skill is stored at 0600 permissions on the operator's local filesystem (no server, no cloud). The loader uses yaml.safe_load() only. The render gate handles no sensitive data. TLS for outbound Anthropic API calls is handled by the claude CLI subprocess. PASS.

**A03 Injection:**

- render_gate.py regex (SEC-12 focus): The _ANCHOR_RE pattern in render_gate.py is a fixed compiled regex applied to operator-supplied markdown (the drafted report). Patterns contain no nested quantifiers or alternation within quantifiers that would cause catastrophic backtracking. The longest alternation branch (the Branding Header pattern) uses `[^*]+` inside `\*\*[^*]+\*\*` — possessive-like but not catastrophic under PCRE or Python re. Input is operator-drafted markdown piped via stdin; no network-supplied or untrusted-user-supplied input. INFORMATIONAL — not a real ReDoS risk in this operator-only context.

- mktemp template in portfolio-checkup SKILL.md: Fixed at commit 5a36d36. The current SKILL.md:108 uses `mktemp "${TMPDIR:-/tmp}/pcheckup.XXXXXX"` (6 trailing X's, properly randomized). The prior non-randomizing `.md`-suffix bug is gone. Holdings are written to the temp file via `cat > "$DRAFT" <<'REPORT'` (heredoc with quoted delimiter — no variable expansion in body). Holdings flow as stdin data to render_gate.py, never into the shell command string. PASS.

- white-label loader.py: Uses yaml.safe_load() (not yaml.load()); no pickle, no eval(), no exec(). Logo URL path handling at loader.py:940 uses urllib.parse.urlparse() for host extraction only — no external network fetch in the loader itself (URL fetching is in extract/web_pdf.py, which is the extraction step, not the load step). PASS.

- Path traversal in eval_spec.load_spec(): Unchanged from prior audit. Skill parameter is operator-controlled (hardcoded or --skill flag); not user- or network-supplied. INFORMATIONAL — acceptable for internal developer tool; would require remediation if exposed via HTTP.

- importlib.exec_module on eval_config.py: Unchanged from prior audit. Repo-local developer-authored files only. INFORMATIONAL.

**A10 SSRF:** The white-label extraction layer (extract/web_pdf.py) makes outbound HTTP calls to operator-supplied URLs for branding extraction. This is the extraction step (skill-invocation time, operator-supplied URL), not the loader. The URL is supplied by the operator, not a remote user. No server exists to be a SSRF intermediary — this is operator-local tooling. Same classification as prior audit's web_pdf.py finding: INFORMATIONAL — no server, no SSRF threat model applicable.

**Attacker Perspective (merged surface additions):**
1. render_gate.py — can crafted input prevent report delivery? The fail-safe (no anchor found → return input unchanged) ensures the gate never destroys a report. An attacker would need to strip all rendered headers from the draft, which would require controlling the LLM output upstream. Not an incremental attack surface.
2. white-label branding — can a malicious config.yaml inject commands? The loader uses yaml.safe_load() and treats the config as data only (no eval, no exec). Downstream consumers read color tokens and font names as strings — no code execution path from config values.
3. mktemp temp file — race condition on /tmp? The fix (XXXXXX template) generates a random 6-char suffix via mktemp's OS-level atomic creation. World-writable /tmp symlink attack requires knowing the exact path before creation — randomized template closes this. PASS.
